# ChaoJia

`ChaoJia` 是一个多 AI 聚合聊天扩展。

它的核心思路不是走 API，也不是让用户自己填 `API Key`，而是在同一个界面里把多个已经登录的 AI 网页会话并排打开，再把右侧网页中的对话内容实时聚合到左侧统一聊天区里。这样你可以在一个地方同时观察、记录和整理多个 AI 的回答。

## 核心特点

- 一个界面同时和多个 AI 对话
- 不需要自己配置 `API Key`
- 不直接消耗你自己的 API 配额
- 右侧保留原始网页会话，左侧统一聚合结果
- 支持文字、代码、链接、图片等内容聚合显示
- 支持把聚合后的聊天内容导出为离线资料

## 它是怎么工作的

`ChaoJia` 会在扩展页面右侧嵌入多个 AI 官方网页。

你在左侧输入一次问题后，扩展会把同一条消息发送到当前启用的多个网页会话中。等这些网页返回回答后，扩展再把它们的内容抓取并汇总到左侧统一视图里。

这个机制有几个直接好处：

- 你不用来回切多个站点
- 你可以很直观地比较不同 AI 的回答
- 你能把一次调研过程完整沉淀下来
- 你依然在使用网页端自己的会话环境

## 和 API 方案的区别

很多聚合工具的前提是：

- 你得自己申请 `API Key`
- 你得自己承担 token 成本
- 你得处理不同平台的接入差异

`ChaoJia` 选择的是另一条路：

- 不要求用户提供 `API Key`
- 直接复用你已经登录的网页端会话
- 把网页中的实际回答结果聚合回来

这使得它更适合日常资料收集、横向对比、写作辅助和轻量研究场景。

说明：

- 具体可用性仍然取决于对应 AI 网页本身的登录状态、权限、频率限制和服务策略。
- 本项目不会绕过服务方的限制，也不保证所有网站结构长期不变。

## 适合的使用场景

- 同时问 `ChatGPT`、`Gemini` 等多个 AI
- 对比多个模型对同一问题的回答差异
- 收集资料、整理灵感、沉淀研究过程
- 保存带图片的聊天记录作为离线文档
- 长时间积累自己的 AI 调研资料库

## 导出能力

`ChaoJia` 支持把当前聚合聊天导出成离线文档包。

导出结果包括：

- `chat.md`
- 聊天中引用到的图片
- 对应的本地文件路径引用

这样你导出的内容不只是纯文本，而是一份可以离线查看、继续整理、继续归档的 Markdown 资料。

这对于以下场景非常方便：

- 做研究资料沉淀
- 收集网页答案和配图
- 后续二次整理到知识库
- 发给别人做协作或留档

## 当前方向

目前项目重点在于：

- 多供应商聊天聚合
- 历史会话管理与搜索
- 图片抓取与展示优化
- 导出为带图片的离线 Markdown 文档
- 更稳定的网页端适配能力

## 开发

安装依赖：

```bash
npm install
```

开发构建：

```bash
npm run dev
```

生产构建：

```bash
npm run build
```

在浏览器扩展管理页中加载：

```text
/Users/tetsuya/Development/chaojia/dist
```

## 提醒

这是一个基于网页会话聚合的工具，因此目标网站一旦改版，部分选择器、抓取逻辑或显示结构可能需要同步调整。
