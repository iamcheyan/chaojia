# Debug Session: chatgpt-history-images

Status: [OPEN]

## Symptom
- 历史记录里的 `ChatGPT` 图片在左侧仍然按原始大图/乱布局显示。
- 用户希望图片像 `Gemini` 那样以较小卡片横排展示，数量多时自动换行。

## Expected
- 历史记录中的旧消息和新消息都应使用统一的小图卡片布局。
- 对于 `ChatGPT` 返回的“多层包装 div + 多张 img + 后续文本”的结构，左侧应先聚合图片，再在下方保留正文。

## Hypotheses
- H1: 历史记录渲染确实进入了 `layoutMessageImages()`，但没有命中 `ChatGPT` 这类容器结构。
- H2: 图片分组在错误的 DOM 层级执行，导致更深层先处理掉了单张图，外层无法再把兄弟图片合并成一排。
- H3: `sanitizeAiHtml()` 清洗后的历史消息 DOM 与用户贴出的原始结构不同，导致分组条件失效。
- H4: 历史记录里的图片节点已经被改写进布局容器，但 CSS 选择器没有覆盖到实际结构。

## Plan
- 启动调试服务器，记录历史消息渲染时的 DOM 结构和图片分组结果。
- 让用户只通过打开历史记录复现一次，不需要继续发送新消息。
- 基于日志确认是“未命中分组”还是“命中后被 CSS 漏掉”，再做最小修复。
